## 全自动知识卡片元数据生成规范 (v1.0)

### 全局原则

- 所有字段必须基于正文内容推断，禁止凭空编造。
- 输出的语言默认与正文一致；专有名词保留英文原样。
- 每个字段的输出必须精炼、无前缀、无包装，可直接填入 YAML。

---

### 字段定义与生成标准

#### aliases — 文件别名

- 用途：机器可读的短标识，用作文件名或内部链接。
- 生成规则：
    - 提取 2~5 个最能代表核心内容且辨识度高的关键词。
    - 用下划线连接，按重要性从高到低排列。
    - 专业术语/专有名词保留英文大小写原形（如 BERT），英文词组内空格转下划线（如 `Hugging_Face`）。
    - 严格禁止使用“笔记”“报告”“指南”“应用”“实践”等泛义词。
- 示例：
    - 正文讲 Python Pandas 数据分析 → `Python_Pandas_数据分析`
    - BERT 中文情感分析调优 → `BERT_中文_情感分析_调优`

#### description — 概念性摘要

- 用途：让读者一眼知道文章“讲什么”，不透露具体结论。
- 生成规则：
    - 仅说明主题、范围和涉及的话题领域。
    - 禁止出现具体结论、数据、方法或洞见。
    - 禁止使用“本文”“这篇文章”“本章”等指代性短语。
    - 不超过 50 字（英文 160 字符）。
- 示例：
    - Serverless 冷启动的成因、影响及常见优化手段概览。

#### summary — 精华要点

- 用途：读者读完后应记住的核心收获，可能是结论、方法、决定或行动项。
- 生成规则：
    - 用中文分号“；”分隔的列表，无序号、无换行。
    - 3~5 条要点，每条不超过 25 字。
    - 必须是实质信息（结论、洞见、关键决策），而不是话题描述。
    - 禁止“本文”“作者”等主观词。
- 示例：
    - 冷启动延迟主要来自容器初始化和代码加载；使用轻量运行时可将启动降至百毫秒；实例复用比单纯预留并发更节省成本。

#### next_step — 后续行动建议

- 用途：一句前瞻性建议，告诉读者下一步可以看什么、做什么。
- 生成规则：
    - 输出一句直接的建议，不超过 25 字（英文 80 字符）。
    - 不概括当前文档，不用“您可以”“我们建议”等委婉语，直接表达。
    - 可推荐关联主题、实践任务或延伸阅读。
- 示例：
    - 接下来可探索预留并发策略或阅读《事件驱动架构设计》。

#### problem — 核心问题

- 用途：以问句形式表达文档试图解决的核心疑惑。
- 生成规则：
    - 输出 1~3 个简洁的独立问句，用换行分隔，不加序号。
    - 每个问题 ≤ 25 字（英文 80 字符）。
    - 只提问，不回答，不概括内容。
- 示例：
    - 冷启动的根源是什么？
    - 如何在生产环境中有效降低冷启动延迟？

#### related_concept — 关联概念

- 用途：与内容强关联的外部发散概念，用于知识网络连接。
- 生成规则：
    - 最多 5 个概念，用正斜杠“/”分隔，无任何解释。
    - 概念必须是外部邻近思想，不是文档内部提炼（如 `冷启动` 太直接，应选 `事件驱动架构 / 分布式追踪`）。
    - 每个概念尽量 1~3 个词。
- 示例：
    - 事件驱动架构 / 分布式追踪 / 垃圾回收 / 边缘计算

#### category — 归档分类

- 用途：将文档归入一个或多个预定义类别。
- 类别列表与含义：
    - 生活记录：日常活动、旅行、购物、美食、娱乐等。
    - 健康：身体、心理健康的严肃记录和思考。
    - 情绪与思考：心情、感悟、个人世界观等内在体验。
    - 阅读笔记：已读书籍/文章的摘录或读后感。
    - 待读清单：计划阅读的论文、书籍等。
    - 学习笔记：系统学习的知识、方法论、课程笔记。
    - 创意与创作：灵感、写作草稿、绘画、设计等创作过程。
    - 专业技术：技术方案、开发笔记、故障复盘等工程实践。
    - 产品与设计：产品分析、UX 思考、设计文档。
    - 项目与会议：项目文档、会议纪要、日报周报等。
    - 求职面试：面试经验、题目整理。
    - 演讲与分享：讲稿、演示、对外分享材料。
    - 财务：投资记录、记账、财务规划。
    - 任务管理：待办事项、行动清单。
    - 个人信息：个人资料、档案。
    - 其他：无法归入以上类别的零散内容。
- 生成规则：
    - 优先选最专指类别，不选宽泛类别。例如已属“专业技术”就不选“学习笔记”。
    - 可多选（≤3 个），用“/”分隔。
    - 无匹配时选“其他”。
- 示例：
    - 一篇用技术方案做的产品分析 → 专业技术/产品与设计

#### suggestion — 完善建议

- 用途：基于类别和内容完整度，给出让笔记更完善的建议。
- 内置类别知识：生成时需参考上述 category 的完整列表与含义，站在该类别最佳实践角度提建议。
- 生成规则：
    - 输出 1~2 条具体可操作建议，用中文分号“；”分隔，每条 ≤ 25 字。
    - 建议方向：补充背景/结论/数据/示例/参考资料、明确模糊情绪、增加行动项等。
    - 不出现类别名称本身，直接给出行动指导。
    - 若内容已很完整，输出“内容已较完整，无需补充”。
- 示例：
    - 补充明确负责人和截止时间；增加待决议题的背景说明。

#### tags — 主题标签

- 用途：高度抽象的嵌套标签体系，用于主题聚合。
- 根标签体系（只能使用这些，不能新增）：
    - 编程与架构：编程语言、软件设计、系统架构、代码实现等。
    - 算法与模型：算法设计、数据结构、数学模型、优化方法等。
    - 工具与实践：软件工具、框架使用、配置技巧、效率方法。
    - 数学与逻辑：纯数学、统计学、逻辑学、计算理论。
    - 人工智能：机器学习、深度学习、NLP、CV、强化学习等。
    - 产品与设计：产品策略、用户体验、交互设计、服务设计。
    - 项目管理：项目规划、进度管理、团队协作、敏捷方法。
    - 商业与管理：企业管理、商业模式、市场营销、创业经营。
    - 职业发展：职场技能、面试求职、晋升策略、软技能。
    - 领域知识：特定行业的专业知识、术语、法规、流程。
    - 阅读与写作：读书笔记、文章写作、创作技巧、文本分析。
    - 学习方法：学习策略、记忆方法、应试技巧、知识内化。
    - 教育：教学理论、课程设计、教育政策、教育技术。
    - 信息管理：知识管理、笔记方法、个人信息整理、PKM。
    - 健康：身体、心理健康、疾病、医疗等严肃内容。
    - 情绪与心理：情绪体验、心理状态、性格分析、人际互动。
    - 运动与饮食：健身、运动训练、食谱、营养、饮食文化。
    - 自我管理：个人习惯、时间管理、自律方法、人生规划。
    - 旅行与自然：旅行经历、自然风景、地理人文、户外探险。
    - 创作与设计：绘画、写作、平面设计、视频创作等。
    - 影视与音乐：电影、剧集、纪录片、音乐作品、评论。
    - 财务与经济：投资理财、个人财务、经济分析、税务规划。
    - 人文与社科：历史、文学、社会学、人类学、政治学等。
    - 哲学与思维：哲学思辨、思维模型、批判性思维、认知科学。
    - 自然科学：物理、化学、生物、天文、地球科学等。
    - 其他：无法归入的杂项。
- 生成规则：
    - 优先选 1~3 个根标签。若需进一步具象化，输出“根标签/具体概念”。具体概念必须为高热度、通用抽象词（≤3 字或一个英文单词），如“编程与架构/微服务”。
    - 禁止创造新根标签；若无合适根标签，用“其他”。
    - 总标签数 ≤ 5，用英文逗号“,”分隔。
- 示例：
    - 编程与架构/微服务, 工具与实践/Docker, 项目管理

#### audience — 目标读者画像

- 用途：谁是这篇文档最适合的读者。
- 生成规则：
    - 用一句中文短语（≤50 字）描绘一类或多类读者。
    - 可包括身份、关键特征或痛点。多类读者用顿号分隔，前可加共同特征。
    - 若极为通用，输出“普通读者”。
- 示例：
    - 想系统学习系统设计的初级后端工程师
    - 需要优化数据库性能的后端、DBA、架构师

#### difficulty — 阅读难度

- 用途：阅读该文档需要的前置知识门槛。
- 分级标准：
    - 入门：无任何专有术语或复杂概念，生活化表达。
    - 进阶：含少量领域常用词，有基本教育背景即可理解。
    - 专业：术语密集，假定读者有系统知识或 1-3 年领域经验。
    - 专家：探讨前沿、优化极限或罕见情况，需深厚经验。
    - 行业：包含特定行业的法规、行话或认证知识，脱离该行业无法理解。
- 输出：仅输出“入门”“进阶”“专业”“专家”“行业”中的一个。

#### rating — 价值评分

- 用途：该文档作为长期笔记的参考价值。
- 评分标准：
    - 5：极具价值。深刻洞察、罕见经验或高度浓缩的思维框架，值得反复引用。
    - 4：高价值。有清晰结论、实用方法或独特视角。
    - 3：中等价值。内容完整，有实质，但未超出常规认知。
    - 2：低价值。碎片化、临时性、表达不完整或观点模糊。
    - 1：几乎没有留存价值。纯流水账、过时信息、纯宣泄无反思。
- 输出：仅输出一个数字（1-5）。

#### title — 人类可读标题（AI 生成）

- 用途：为文档生成适合人类阅读的完整标题。
- 核心规则：
    - 标题风格**必须**根据文档类别适配，参照下方风格表。
    - 中文 ≤20 字，英文 ≤12 词；禁止出现“笔记”“文档”“报告”“心得”“总结”等类型词。
    - 只输出标题文本，不加任何引号或装饰。

各类别标题风格速查表：

|类别|标题风格要求|冒号用法|
|---|---|---|
|生活记录|时间/地点 + 事件/感受，口语化|不用|
|健康|症状/科目 + 关键信息，直接|可用|
|情绪与思考|一句话提炼核心感触或追问|不用|
|阅读笔记|书名/篇名 + 核心提炼|可用|
|待读清单|主题 + 待读指向|不用|
|学习笔记|知识领域 + 具体知识点|可用|
|创意与创作|项目/主题 + 版本或状态|可用|
|专业技术|技术点/问题 + 解决/方案|可用|
|产品与设计|产品名/功能 + 分析角度|可用|
|项目与会议|会议名/项目 + 日期 + 关键议题|可用|
|求职面试|公司/岗位 + 面试主题|可用|
|演讲与分享|主题 + 受众或目的|可用|
|财务|类型 + 时间/标的|不用|
|任务管理|行动指向，直接任务描述|不用|
|个人信息|档案名称，简洁直接|不用|
|其他|自由概括，简洁为主|视情况|

> **冒号使用准则**：仅当需同时呈现“话题”与“焦点/结论”时才可用冒号，生活记录、情绪思考、任务管理、财务、个人信息类别不鼓励使用。

### 元数据输出格式

请严格按以下顺序输出，每行一个字段，格式为“字段名: 值”：

```
aliases: …
description: …
summary: …
next_step: …
problem: …
related_concept: …
category: …
suggestion: …
tags: …
audience: …
difficulty: …
rating: …
```

只输出上述块，不要任何额外说明。
