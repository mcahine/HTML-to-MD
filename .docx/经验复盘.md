# 经验复盘

## 踩过的坑

### 1. 删除文件必须走回收站

**教训**：`rm -rf 中国政府网` 误删了该目录下全部 HTML 源文件，无法恢复。

**规则**：所有删除操作使用 `send2trash`，全局配置文件 `~/.claude/CLAUDE.md` 已记录。

### 2. 转换器输出目录混乱

**问题**：部分转换器将 MD 输出到硬编码子目录（如 `财联社/`），但 HTML 已在同名目录中，导致 `财联社/财联社/file.md`。

**解决**：batch 脚本统一从项目根目录调用，`output_dir='.'`，转换器子目录名恰好匹配 HTML 目录名。

### 3. 图片附件存放方式不统一

**问题**：7 个转换器将图片直接放在 `assets/` 下（扁平结构），不同文章的图片混在一起，引用混乱。

**解决**：统一使用 base class 的 `_extract_base64_images`，自动按 `assets/{stem}_assets/image_N.ext` 分目录存放。

### 4. data:image 残留

**问题**：部分转换器先 deepcopy 了 content 再提取图片，导致 img src 的修改不反映到 markdown 输出中。

**解决**：在 `_html_to_markdown` 之前调用 `_extract_base64_images`，确保 soup 和 markdown 使用同一对象。polish 脚本增加了 `fix_data_uri_images` 作为兜底。

### 5. 中国政府网与国家发展改革委域名混淆

**问题**：两者都是 `.gov.cn`，早期按文件名后缀分类，混杂在一起。

**解决**：按域名精确区分——`zfxxgk.ndrc.gov.cn` → 国家发展改革委，`www.gov.cn` → 中国政府网。

### 6. cls.cn 域名下的多来源

**问题**：财联社（cls.cn）的文章在文件名中标注为北京日报、科创板日报、澎湃新闻等不同来源，被错误分类到独立目录。

**解决**：按 HTML 内的实际域名（都是 cls.cn）统一归入财联社。

### 7. Windows 换行符导致正则失效

**问题**：文件使用 `$$$$$$$$\r$$$$$$$$$$$$$$$$\n` 换行，但清理代码的正则只匹配 `\n$$$$$$$$$$$$$$$$\n$$$$$$$$`，导致代码块空行清理等规则无效。

**解决**：在 `fix_code_block_spacing` 开头统一归一化换行符。

### 8. 加粗标记与文字间有空格

**问题**：部分转换器产生 `**text**` 或 `**text**`，Obsidian 无法渲染为加粗。

**解决**：polish 脚本增加 `fix_spaced_bold`，使用精确匹配去除加粗标记内侧空格。

### 9. 文件名截断

**问题**：部分文件名过长被截断（如 `..._2025122` 缺少最后一位），日期格式不标准。

**解决**：创建 `rename_interactive.py`，根据 MD 的 YAML 元数据辅助人工重命名。

### 10. 未引用图片占用空间

**问题**：多次转换后，旧的文章名对应的 assets 子目录仍保留，累计占用 >1 GB。

**解决**：`cleanup_unused_images.py` 通过 MD 引用路径精确匹配，安全清理。
