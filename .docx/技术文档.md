# 技术文档

## 核心模块

### html_converter_base.py — 基础转换器

所有转换器的基类。提供：

- `convert(html_path, output_dir)` — 标准转换流程
- `_extract_base64_images(soup, html_path, output_dir)` — 提取 Base64 图片到 `assets/{stem}_assets/`
- `_build_yaml_front_matter(metadata)` — 生成统一的 YAML 头
- `_html_to_markdown(content)` — html2text 转换
- `batch_convert(input_dir, output_dir)` — 批量转换

### batch_convert_all.py — 批量转换入口

- 注册所有网站目录与转换器的映射（`CONVERTERS` 列表）
- 按目录遍历，对每个 HTML 调用对应转换器
- 处理输出目录名称不匹配（如 `mp_weixin_qq_com` → `微信公众号`）
- 输出统计：成功数、失败数

### polish_markdown.py — MD 后处理

统一后处理流水线，修复转换器产生的格式问题：

| 函数 | 功能 |
|------|------|
| `normalize_yaml` | 统一 YAML 字段，移除非标准键 |
| `fix_backslash_paths` | Windows 反斜杠 → 正斜杠 |
| `fix_spaced_bold` | `**text**` → `**text**`（加粗标记空格） |
| `fix_javascript_void_links` | 移除 TOC 锚点链接 |
| `fix_tipster_line` | IT之家线索行加 `>` 引用 |
| `fix_source_date_junk` | 移除泄漏的日期/来源行 |
| `fix_data_uri_images` | data:image → 本地 assets 路径 |
| `fix_indented_code_blocks` | 4空格缩进代码 → ``` 围栏 |
| `fix_list_spacing` | 合并同类型列表项间的空行 |
| `fix_code_block_spacing` | 去除代码块内首尾空行 |
| `fix_heading_spacing` | 标题前后空行规范化 |
| `fix_bbcode_markers` | 移除 `[code]`/`[/code]` 标记 |
| `fix_bare_urls` | 裸 URL → Markdown 链接 |

### cleanup_unused_images.py — 清理未引用图片

逻辑：
1. 收集每个网站所有 MD 中的 `

![alt](path)

` 引用路径
2. 扫描 assets/ 和 images/ 下的所有文件
3. 计算文件相对于网站目录的路径
4. 如果路径不在引用集合中 → 未引用 → 删除（回收站）

### archive_html.py — HTML 归档

将所有网站子目录中的 .html 文件移动（非复制）到同级 `Project-xxx-副本` 目录，保持目录结构。

### rename_interactive.py — 交互式重命名

- 默认模式：逐文件展示元数据，等用户输入新名
- `--list`：JSON 格式输出待重命名文件列表
- `--rename "dir/stem" "new_stem"`：CLI 单文件重命名
- `--batch`：stdin 批量（Tab 分隔）

### safe_delete.py — 安全删除

封装 `send2trash`，文件和目录都走回收站。
