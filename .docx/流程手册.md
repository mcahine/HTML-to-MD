# 流程手册

## 日常新增网页的处理流程

### 第一步：分类

将新下载的 HTML 文件放到项目根目录，然后根据域名分类：

```bash

# 手动分类：看文件名后缀（如 _财联社.html → 财联社/）

# 或自动：根据 HTML 内的 og:url / canonical link 确定域名

分类规则：
- 已存在的网站目录 → 移入对应目录
- 新网站 → 创建以域名命名的目录
- 文件名后缀只是辅助标识，最终以域名分类为准

### 第二步：转换

bash
python 整合成果/batch_convert_all.py

如有新网站：
1. 在 `整合成果/converters/` 创建 `convert_xxx.py`
2. 继承 `HTMLConverterBase`，设置 CSS 选择器
3. 在 `batch_convert_all.py` 的 `CONVERTERS` 列表添加条目

### 第三步：后处理

```bash
python 整合成果/polish_markdown.py

检查是否有杂信息残留（来源行、日期行、下载链接等）。如有，在对应转换器或 polish 脚本中添加清理规则。

### 第四步：清理未引用图片

bash

# 先预览

python 整合成果/cleanup_unused_images.py

# 确认无误后执行

python 整合成果/cleanup_unused_images.py --do

### 第五步：归档 HTML

```bash
python 整合成果/archive_html.py

HTML 文件将移至同级 `Project-xxx-副本` 目录。

## 验证清单

- [ ] 所有 HTML 都有对应 MD
- [ ] MD 中的图片引用指向存在的文件
- [ ] 没有 `data:image` 残留
- [ ] YAML 头包含 title/author/date/source/url/converted_at
- [ ] 文件命名符合 `关键词_YYYYMMDD_来源` 格式
- [ ] 加粗标记 `**text**` 内外无空格
